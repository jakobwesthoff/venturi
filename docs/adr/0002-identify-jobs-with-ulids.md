# 2. Identify jobs with ULIDs

Date: 2026-06-07

## Status

Accepted

## Context

Every job needs a unique identifier that exists before the row is persisted, so
a producer can refer to a job at the moment it enqueues it. ULIDs are 128-bit
identifiers that encode their creation timestamp in the high bits, are therefore
lexicographically sortable by creation order, and render as a 26-character
Crockford base32 string. They can be generated by the producer at enqueue time,
before any database round-trip, and stored as text.

## Decision

venturi identifies each job with a ULID, generated by the producer at enqueue
time. The concrete storage representation (textual form versus a binary or UUID
column) is deferred to the schema-design document.

## Consequences

Identifiers are client-generated, so an enqueue call can return the id without
reading the row back. Sorting on the id yields creation order and is available
as a tiebreaker. venturi takes a dependency on a ULID implementation.
